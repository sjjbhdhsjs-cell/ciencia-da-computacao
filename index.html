1. A História da Computação

A jornada da computação é marcada por uma evolução exponencial, desde dispositivos mecânicos colossais até os microchips que carregamos em nossos bolsos. Compreender essa história é entender a base de toda a tecnologia moderna.

Dispositivos Mecânicos

A necessidade de calcular é antiga. O Ábaco (c. 2700 a.C.) foi um dos primeiros dispositivos de contagem. Séculos depois, no século XIX, Charles Babbage projetou a "Máquina Analítica", um conceito revolucionário de um computador mecânico programável. Ada Lovelace, ao escrever algoritmos para essa máquina teórica, tornou-se a primeira programadora da história.

As Gerações de Computadores

  • 1ª Geração (1940-1956): Válvulas. Marcada por máquinas como o ENIAC. Eram gigantescos, consumiam muita energia e usavam válvulas eletrônicas para processamento. A programação era feita diretamente em linguagem de máquina.
  • 2ª Geração (1956-1963): Transistores. A invenção do transistor substituiu as válvulas, tornando os computadores menores, mais rápidos, mais baratos e mais confiáveis. Surgem as primeiras linguagens de alto nível, como FORTRAN e COBOL.
  • 3ª Geração (1964-1971): Circuitos Integrados (CIs). O desenvolvimento do CI (ou microchip) permitiu que centenas de transistores fossem colocados em um único chip de silício. Isso levou ao surgimento dos minicomputadores e dos primeiros Sistemas Operacionais como os conhecemos.
  • 4ª Geração (1971-Presente): Microprocessadores. A criação do microprocessador (como o Intel 4004) colocou uma CPU inteira em um único chip. Isso possibilitou a revolução do computador pessoal (PC), com o lançamento do Apple II e do IBM PC.
  • 5ª Geração (Presente e Futuro): Focada em processamento paralelo, Inteligência Artificial e, mais recentemente, na Computação Quântica, que promete resolver problemas hoje intratáveis.

O termo "bug" (inseto) para descrever uma falha de sistema foi popularizado por Grace Hopper. Em 1947, técnicos encontraram uma mariposa (moth) presa em um relé do computador Mark II, causando um mau funcionamento. O inseto foi removido e colado no diário de bordo com a anotação: "Primeiro caso real de bug sendo encontrado."

2. Representação de Dados

Computadores não entendem palavras, imagens ou sons. Eles entendem apenas eletricidade: ligado ou desligado. Toda a complexidade do mundo digital é construída sobre um sistema simples de dois estados: 0 (desligado) e 1 (ligado).

Bits e Bytes

  • Bit (Dígito Binário): É a menor unidade de informação. Pode ser 0 ou 1.
  • Byte: Um conjunto de 8 bits. É a unidade básica usada para representar caracteres. Com 8 bits, podemos criar $2^8$, ou 256, combinações diferentes.

Sistemas Numéricos

Usamos o sistema Decimal (base 10) em nosso dia a dia. Os computadores usam o Binário (base 2). Para facilitar a leitura de longas sequências binárias, programadores usam o Hexadecimal (base 16), que usa os dígitos 0-9 e as letras A-F.

Exemplo: O número decimal 173 é 10101101 em binário e AD em hexadecimal.

Codificação de Caracteres

Como um número representa uma letra?

  • ASCII (American Standard Code for Information Interchange): O padrão original. Usa 7 bits (expandido para 8 bits) para mapear números a letras (ex: 'A' é 65, 'a' é 97), números e símbolos de pontuação.
  • Unicode (UTF-8): O ASCII era limitado e não cobria idiomas com acentos ou alfabetos não latinos. O UTF-8 é um padrão universal que pode representar (quase) todos os caracteres de todos os idiomas do mundo, usando uma quantidade variável de bytes. É o padrão dominante na web hoje.

Representação de Imagens e Sons

  • Imagens (Bitmap): A imagem é dividida em uma grade de pixels. Para cada pixel, o computador armazena um valor de cor (ex: o sistema RGB, que armazena a intensidade de Vermelho, Verde e Azul).
  • Sons (Digitalização): O som é uma onda analógica contínua. Para digitalizá-la, medimos a amplitude (altura) da onda milhares de vezes por segundo (taxa de amostragem) e armazenamos o valor de cada medição.

3. Arquitetura de Computadores

A arquitetura define como os componentes de hardware são organizados e interconectados para criar um sistema funcional. A base da maioria dos computadores modernos é a Arquitetura de von Neumann.

Componentes Principais

  • CPU (Unidade Central de Processamento): O "cérebro" do computador. Executa instruções e realiza cálculos.
  • Memória (RAM): Armazena temporariamente as instruções e os dados que a CPU está usando no momento. É volátil (perde dados quando desligada).
  • Dispositivos de E/S (Entrada/Saída): Permitem a interação com o mundo exterior (teclado, mouse, monitor, SSD/HD).
  • Barramento (Bus): O sistema de comunicação que conecta todos esses componentes, transportando dados, endereços e sinais de controle.

Dentro da CPU

A CPU é composta por várias partes:

  • ULA (Unidade Lógica e Aritmética): Realiza as operações matemáticas (soma, subtração) e lógicas (E, OU, NÃO).
  • Unidade de Controle (UC): Busca as instruções na memória, decodifica-as e gerencia a execução, coordenando a ULA e outros componentes.
  • Registradores: Pequenas unidades de memória ultrarrápidas dentro da própria CPU, usadas para armazenar temporariamente o dado ou instrução que está sendo processado no exato momento.

Hierarquia de Memória

Para otimizar a performance, a memória é organizada em níveis. Quanto mais rápido, menor e mais caro:

  1. Registradores (na CPU): Mais rápido.
  2. Cache (L1, L2, L3): Memória rápida que armazena cópias de dados da RAM usados frequentemente.
  3. RAM (Memória Principal): A memória de trabalho.
  4. Armazenamento Secundário (SSD, HD): Mais lento, mas permanente e de grande capacidade.

4. Sistemas Operacionais (SO)

O Sistema Operacional (Windows, macOS, Linux, Android) é o software mais importante do computador. Ele atua como um intermediário entre o usuário e o hardware, gerenciando todos os recursos da máquina.

Funções Principais

  • Abstração de Hardware: O SO esconde a complexidade do hardware dos programas. Um desenvolvedor não precisa saber *como* gravar dados em um SSD; ele apenas "pede" ao SO para salvar um arquivo.
  • Gerenciamento de Recursos: O SO decide qual programa usa a CPU, quanta memória cada um recebe e quem pode acessar dispositivos.
  • Interface do Usuário (UI): Fornece a maneira como interagimos com a máquina, seja através de uma Interface Gráfica (GUI) ou uma Linha de Comando (CLI).

Processos e Threads

Quando você abre um programa (como um navegador), o SO cria um Processo. Este processo é um ambiente isolado com sua própria memória.

Dentro de um processo, podem existir várias Threads (linhas de execução). Por exemplo, em um navegador, uma thread desenha a página, outra baixa uma imagem e uma terceira toca uma música. As threads permitem que um programa faça várias coisas "ao mesmo tempo" (concorrência).

Gerenciamento de Memória

O SO garante que o Processo A não possa ler ou alterar a memória do Processo B (isolamento). Ele também usa uma técnica chamada Memória Virtual, onde parte do disco rígido (HD/SSD) é usada como uma extensão da RAM, permitindo que o sistema execute mais programas do que caberiam na memória física.

Sistemas de Arquivos

O SO organiza os dados no disco em uma estrutura de diretórios (pastas) e arquivos. Ele controla quem pode ler, escrever ou executar arquivos (permissões). Exemplos de sistemas de arquivos incluem NTFS (Windows), APFS (Apple) e EXT4 (Linux).

5. Algoritmos e Estruturas de Dados

Estes são os dois pilares da programação. Um algoritmo é uma receita, um conjunto finito de passos para resolver um problema. Uma estrutura de dados é uma forma organizada de armazenar e acessar dados para que possam ser usados eficientemente.

Algoritmos Fundamentais

  • Busca (Search): Encontrar um item. A Busca Linear verifica um por um. A Busca Binária (muito mais rápida) só funciona em dados ordenados e divide o problema pela metade a cada passo.
  • Ordenação (Sort): Organizar dados. Bubble Sort é simples, mas lento (compara pares). Merge Sort e Quick Sort são muito mais eficientes (usam a estratégia de "dividir para conquistar").

Complexidade (Notação Big O)

Não basta um algoritmo funcionar; ele precisa ser eficiente. A Notação Big O descreve o "pior caso" de um algoritmo, ou como seu tempo de execução cresce à medida que a quantidade de dados ($n$) aumenta.

  • $O(1)$ (Constante): Excelente. O tempo é o mesmo, não importa o tamanho.
  • $O(\log n)$ (Logarítmico): Ótimo. (ex: Busca Binária).
  • $O(n)$ (Linear): Bom. O tempo cresce na mesma proporção que os dados. (ex: Busca Linear).
  • $O(n \log n)$ (Log-Linear): Ótimo. Padrão de algoritmos de ordenação eficientes (Quick Sort, Merge Sort).
  • $O(n^2)$ (Quadrático): Ruim. O tempo cresce exponencialmente. (ex: Loops aninhados for dentro de for, Bubble Sort).

Estruturas de Dados Comuns

  • Array (Vetor): Uma lista simples de itens na memória, acessados por um índice (posição).
  • Pilha (Stack): Segue a regra LIFO (Last-In, First-Out). O último a entrar é o primeiro a sair. Pense em uma pilha de pratos. Usado para o botão "Voltar" ou para o histórico de "Desfazer".
  • Fila (Queue): Segue a regra FIFO (First-In, First-Out). O primeiro a entrar é o primeiro a sair. Pense em uma fila de banco. Usado para gerenciar tarefas de impressão ou processos na CPU.
  • Hash Table (Mapa): Uma estrutura otimizada que usa uma "função de hash" para associar uma Chave a um Valor (ex: associar um CPF a um nome). Permite buscas quase instantâneas ($O(1)$ na média).

6. Linguagens de Programação

Linguagens de programação são as ferramentas que usamos para escrever as instruções (algoritmos) que o computador deve executar. Elas variam em nível de abstração, tipagem e paradigma.

Níveis de Linguagem

  • Linguagem de Máquina: Binário (0s e 1s). A única coisa que a CPU entende diretamente.
  • Assembly: Uma representação textual da linguagem de máquina (ex: ADD, MOV). Ainda de baixo nível, mas legível por humanos.
  • Linguagens de Alto Nível (C++, Python, Java): Usam sintaxe próxima da linguagem humana, escondendo a complexidade do hardware.

Compiladas vs. Interpretadas

  • Compiladas (C++, Java, C#): O código-fonte é traduzido *antes* da execução por um Compilador, gerando um arquivo executável (linguagem de máquina). Geralmente resultam em performance mais rápida.
  • Interpretadas (Python, JavaScript, Ruby): O código-fonte é lido e traduzido linha por linha *durante* a execução por um Interpretador. São geralmente mais flexíveis e fáceis de testar.

Paradigmas de Programação

Um paradigma é um estilo ou "jeito" de programar.

  • Imperativo/Procedural (C, Pascal): O foco está em uma sequência de comandos (procedimentos ou funções) que alteram o estado do programa.
  • Orientado a Objetos (OOP) (Java, C#, Python): O foco está em "Objetos" que agrupam dados (atributos) e comportamentos (métodos) em uma única entidade. Facilita a organização de sistemas complexos através de conceitos como Encapsulamento, Herança e Polimorfismo.
  • Funcional (Haskell, Lisp): O foco está em funções matemáticas puras, evitando estados mutáveis e efeitos colaterais.

Tipagem: Estática vs. Dinâmica

  • Tipagem Estática (Java, C++): O tipo de uma variável (inteiro, string) é verificado em tempo de compilação. Reduz erros em tempo de execução.
  • Tipagem Dinâmica (Python, JavaScript): O tipo de uma variável é verificado em tempo de execução. Oferece mais flexibilidade.

7. Redes de Computadores

Redes são o que conectam computadores, permitindo que eles compartilhem dados. A Internet é a maior rede do mundo, um "sistema de sistemas" que funciona graças a um conjunto de regras (protocolos).

LAN, WAN e a Internet

  • LAN (Local Area Network): Uma rede local, como o Wi-Fi da sua casa ou escritório.
  • WAN (Wide Area Network): Uma rede ampla que conecta várias LANs, muitas vezes cruzando cidades ou países.

Protocolos e o Modelo TCP/IP

Para que computadores de diferentes fabricantes e sistemas operacionais possam conversar, eles precisam seguir regras padronizadas. O modelo TCP/IP é a base da Internet.

  • Camada de Aplicação (HTTP, SMTP): Onde os aplicativos (navegador, cliente de email) operam.
  • Camada de Transporte (TCP, UDP): Gerencia a comunicação ponta-a-ponta.
  • Camada de Internet (IP): Responsável por endereçar e rotear os pacotes de dados pela rede.
  • Camada de Enlace (Ethernet, Wi-Fi): Lida com a transmissão física dos dados no meio (cabos, ondas de rádio).

TCP vs. UDP

Na camada de transporte, temos duas escolhas principais:

  • TCP (Transmission Control Protocol): É confiável. Garante que todos os pacotes cheguem na ordem correta e sem erros (ele pede reenvio se algo se perder). Usado para e-mail, sites (HTTP) e downloads.
  • UDP (User Datagram Protocol): É rápido e simples. Ele "dispara e esquece", não garantindo a entrega ou a ordem. Usado onde a velocidade é mais importante que a perfeição, como em streaming de vídeo, jogos online e chamadas de voz (VoIP).

Como a Web Funciona (HTTP e DNS)

  1. Você digita google.com no navegador.
  2. Seu computador pergunta ao DNS (Domain Name System): "Qual o endereço IP de google.com?".
  3. O DNS responde (ex: 172.217.14.228).
  4. Seu navegador envia uma requisição HTTP (Hypertext Transfer Protocol) para esse IP, pedindo a página.
  5. O servidor do Google responde enviando os arquivos HTML, CSS e JavaScript da página.

8. Bancos de Dados

Um banco de dados é uma coleção organizada de dados, gerenciada por um SGBD (Sistema de Gerenciamento de Banco de Dados). Eles são essenciais para armazenar de forma persistente e eficiente as informações de qualquer aplicação.

Relacionais (SQL)

O modelo dominante por décadas. Os dados são organizados em tabelas (como planilhas) que se relacionam entre si através de "chaves".

  • Exemplos: MySQL, PostgreSQL, SQL Server.
  • Linguagem: Usam SQL (Structured Query Language) para definir, consultar e manipular os dados.
  • Consultas (Queries): O comando SELECT busca dados, INSERT adiciona, UPDATE modifica e DELETE remove. O comando JOIN é usado para combinar dados de múltiplas tabelas (ex: buscar um usuário e seus pedidos).
  • ACID: Garantem transações seguras através das propriedades ACID (Atomicidade, Consistência, Isolamento, Durabilidade).

Não-Relacionais (NoSQL)

Surgiram com a necessidade de lidar com volumes massivos de dados (Big Data) e dados não estruturados (como posts em redes sociais), onde a rigidez do SQL era um gargalo.

  • Tipos e Exemplos:
    • Documentos (MongoDB): Armazena dados em documentos flexíveis (formato JSON).
    • Chave-Valor (Redis): Extremamente rápido, ótimo para cache.
    • Colunares (Cassandra): Otimizado para leitura rápida de grandes volumes.
  • Vantagens: Escalabilidade horizontal (fácil de distribuir em várias máquinas) e flexibilidade de esquema (não é preciso definir todas as colunas antecipadamente).

SQL vs. NoSQL

A escolha depende do problema. SQL é excelente para dados estruturados com relações complexas (ex: um e-commerce, um sistema bancário). NoSQL é ótimo para dados massivos e flexíveis (ex: logs de aplicação, redes sociais, dados de IoT).

9. Engenharia de Software

Engenharia de Software é a disciplina de aplicar princípios de engenharia ao design, desenvolvimento, teste e manutenção de software. É o que transforma a programação (uma atividade solitária) em um processo previsível e colaborativo.

Ciclo de Vida do Software

Um processo estruturado para construir software de qualidade:

  1. Requisitos: Entender o que o software precisa fazer (o problema).
  2. Design (Projeto): Planejar a arquitetura, o banco de dados e as interfaces.
  3. Implementação: Escrever o código.
  4. Testes: Verificar se o software faz o que deveria (verificação) e se atende às necessidades do usuário (validação).
  5. Implantação (Deploy): Colocar o software em produção.
  6. Manutenção: Corrigir bugs e adicionar novas funcionalidades.

Metodologias de Desenvolvimento

  • Cascata (Waterfall): Um modelo linear e rígido. Cada fase deve ser 100% concluída antes de iniciar a próxima. Funciona para projetos com requisitos muito claros e imutáveis, mas é inflexível a mudanças.
  • Metodologias Ágeis (Scrum, Kanban): A abordagem moderna. O desenvolvimento é feito em ciclos curtos (Sprints), entregando valor funcional rapidamente e adaptando-se a mudanças nos requisitos. Valoriza a colaboração com o cliente e a resposta rápida a feedbacks.

Testes de Software

Garantir a qualidade é crucial. Existem diferentes níveis de teste:

  • Teste de Unidade: Testa a menor parte do código (uma função ou método) isoladamente.
  • Teste de Integração: Testa como diferentes partes do código (módulos) funcionam juntas.
  • Teste de Sistema (E2E): Testa o sistema completo, simulando o uso real do usuário.

Qual metodologia é mais adequada para um projeto onde os requisitos mudam constantemente?

10. Inteligência Artificial (IA)

IA é um vasto campo da computação focado em criar sistemas que possam simular capacidades humanas, como aprender, raciocinar e perceber o ambiente.

IA vs. Machine Learning vs. Deep Learning

  • IA (Conceito Amplo): Qualquer técnica que permita a uma máquina imitar o comportamento humano (ex: um simples if/else em um jogo).
  • Machine Learning (ML): Um subcampo da IA. Em vez de programar regras explícitas, o sistema "aprende" padrões a partir de grandes volumes de dados.
  • Deep Learning (DL): Um subcampo do ML. Usa "Redes Neurais Artificiais" profundas (com muitas camadas) para resolver problemas complexos, como reconhecimento de imagem e processamento de linguagem.

Tipos de Machine Learning

  • Aprendizado Supervisionado: O modelo é treinado com dados rotulados (ex: milhares de fotos de gatos rotuladas como "gato" e de cães como "cachorro"). O objetivo é prever o rótulo de dados novos. (Usado para classificação e regressão).
  • Aprendizado Não Supervisionado: O modelo é treinado com dados não rotulados. O objetivo é encontrar padrões e estruturas ocultas nos dados (ex: agrupar clientes com comportamentos de compra semelhantes).
  • Aprendizado por Reforço: O modelo aprende tomando decisões e recebendo recompensas ou punições (ex: IA para jogos de xadrez ou carros autônomos).

Processamento de Linguagem Natural (PLN)

PLN é o campo da IA focado em permitir que computadores entendam e processem a linguagem humana (texto e fala). Os LLMs (Large Language Models), como o GPT, são a vanguarda do PLN, treinados em trilhões de palavras da internet para gerar, traduzir e resumir textos com coesão.

11. Segurança da Informação

Segurança (ou Cibersegurança) foca em proteger sistemas, redes e dados contra acesso não autorizado, ataques ou danos. Baseia-se em três pilares (a "Tríade CIA"):

A Tríade CIA

  • Confidencialidade: Garantir que a informação só seja acessível por pessoas autorizadas (usa Criptografia).
  • Integridade: Garantir que a informação não foi alterada indevidamente (usa Hashing).
  • Disponibilidade: Garantir que o sistema esteja disponível e funcional para usuários legítimos.

Criptografia vs. Hashing

Estes dois conceitos são cruciais e diferentes:

  • Criptografia (Mão Dupla): Transforma dados legíveis (texto puro) em ilegíveis (texto cifrado) usando uma chave. O processo é reversível; quem tem a chave pode descriptografar.
    • Simétrica: Usa a mesma chave para criptografar e descriptografar (rápido, mas difícil de compartilhar a chave).
    • Assimétrica (Chave Pública/Privada): Usa um par de chaves. O que a chave pública criptografa, só a chave privada descriptografa. É a base do HTTPS.
  • Hashing (Mão Única): Transforma dados de qualquer tamanho em uma string de tamanho fixo (hash). É irreversível; não se pode obter os dados originais a partir do hash. Usado para verificar a Integridade de arquivos e para armazenar senhas com segurança.

Autenticação e Autorização

  • Autenticação: Provar quem você é (ex: login e senha).
  • Autorização: O que você tem permissão para fazer depois de autenticado (ex: um usuário pode ler, mas um administrador pode deletar).
  • Autenticação de Dois Fatores (2FA): Aumenta a segurança exigindo duas provas (ex: a senha e um código do celular).

Tipos Comuns de Ataque

  • Phishing: Engenharia social para enganar o usuário e fazê-lo revelar informações confidenciais (ex: email falso de banco).
  • SQL Injection: Um ataque que insere código SQL malicioso em campos de formulários para tentar acessar ou corromper o banco de dados.
  • Negação de Serviço (DDoS): Sobrecarregar um servidor com milhões de requisições falsas para tirá-lo do ar (afeta a Disponibilidade).

12. Ética e Impacto Social

A tecnologia não é neutra. As ferramentas que criamos têm um impacto profundo na sociedade, levantando questões éticas complexas sobre privacidade, viés e responsabilidade.

Privacidade de Dados

Vivemos na era do "Big Data". Empresas coletam vastas quantidades de dados sobre nosso comportamento. A ética da privacidade questiona: Quem é o dono desses dados? Como eles podem ser usados? Leis como a LGPD (Brasil) e o GDPR (Europa) foram criadas para dar aos usuários mais controle sobre suas informações pessoais.

Viés (Bias) em IA

Modelos de Machine Learning aprendem dos dados que fornecemos a eles. Se os dados históricos refletem preconceitos sociais (racismo, sexismo), a IA irá aprender e amplificar esses preconceitos. Isso é perigoso em áreas como contratação, concessão de crédito ou policiamento. A ética em IA foca em criar sistemas justos e transparentes.

Propriedade Intelectual

Como equilibramos o direito de um criador de lucrar com sua obra e o direito da sociedade de acessar informação?

  • Copyright: Proteção padrão para obras criativas (código, música, texto).
  • Patentes: Proteção para invenções e processos.
  • Software Livre (Open Source): Um movimento que defende que o código-fonte deve ser aberto, permitindo que qualquer pessoa possa usá-lo, modificá-lo e distribuí-lo. Promove a colaboração e a transparência.

O Futuro do Trabalho

A automação e a IA estão mudando o mercado de trabalho. Enquanto algumas tarefas repetitivas são eliminadas, novas funções (como engenheiros de IA, analistas de dados) são criadas. O desafio ético é como gerenciar essa transição e garantir que os benefícios da tecnologia sejam amplamente distribuídos.

13. Guia de Estudo Exaustivo para Exames

Esta seção foi reestruturada como um guia completo, cobrindo os conceitos essenciais da Ciência da Computação em nível de detalhe exigido por provas de faculdade e exames técnicos.

I. Fundamentos Teóricos e Arquitetura

1. Sistemas de Numeração e Representação de Dados
  • Binário (Base 2): O sistema fundamental do hardware, usando apenas 0 e 1.
  • Decimal (Base 10): Nosso sistema numérico padrão.
  • Hexadecimal (Base 16): Usado por programadores para simplificar longas strings binárias. Usa 0-9 e A-F. (Ex: 11111010 em binário é FA em hexadecimal).
  • Complemento de Dois: O método padrão para representar números inteiros negativos em binário. Para encontrar o negativo de um número: 1) Inverta todos os bits. 2) Some 1. (Ex: 0101 (5) -> 1010 -> 1011 (-5)).
  • Ponto Flutuante (IEEE 754): O padrão para representar números reais (com casas decimais). O número é dividido em três partes: 1) O Sinal (positivo ou negativo). 2) O Expoente (a magnitude do número). 3) A Mantissa (os dígitos significativos do número).
2. Álgebra Booleana e Portas Lógicas

A base matemática dos circuitos digitais. Variáveis são verdadeiro (1) ou falso (0).

  • AND (E): A saída é 1 apenas se ambas as entradas forem 1.
  • OR (OU): A saída é 1 se pelo menos uma das entradas for 1.
  • NOT (NÃO): Inverte a entrada (1 vira 0, 0 vira 1).
  • XOR (OU Exclusivo): A saída é 1 apenas se as entradas forem diferentes.
3. Arquitetura de Computador (Modelo de von Neumann)
  • Ciclo de Instrução: O processo básico da CPU: 1) Buscar a instrução da memória. 2) Decodificar a instrução. 3) Executar a instrução.
  • Componentes da CPU:
    • Unidade de Controle (UC): Gerencia o ciclo de instrução.
    • Unidade Lógica e Aritmética (ULA): Realiza os cálculos (soma, E, OU).
    • Registradores: Pequenas unidades de memória ultrarrápidas dentro da CPU para dados imediatos.
  • Gargalo de von Neumann: A limitação de desempenho causada pelo fato de que a CPU e a Memória Principal (RAM) são separadas e conectadas por um barramento (bus) mais lento. A Memória Cache (L1, L2, L3) existe para mitigar isso.
  • RISC vs. CISC: Duas filosofias de design de CPU. CISC (Intel x86) tem um conjunto grande de instruções complexas. RISC (ARM) tem um conjunto pequeno de instruções simples e rápidas.

II. Estruturas de Dados e Análise de Algoritmos

1. Análise de Complexidade (Notação Big O)

Mede como o tempo de execução (ou uso de memória) de um algoritmo cresce com o aumento da entrada ($n$).

  • $O(1)$ (Constante): Tempo de execução não muda com a entrada. (Ex: Acessar um índice de array array[5]).
  • $O(\log n)$ (Logarítmico): Excelente. O tempo cresce muito lentamente. (Ex: Busca Binária).
  • $O(n)$ (Linear): Bom. O tempo cresce proporcionalmente à entrada. (Ex: Loop for simples).
  • $O(n \log n)$ (Log-Linear): Ótimo. Padrão de algoritmos de ordenação eficientes (Quick Sort, Merge Sort).
  • $O(n^2)$ (Quadrático): Ruim. O tempo cresce exponencialmente. (Ex: Loops aninhados for dentro de for, Bubble Sort).
2. Estruturas de Dados Fundamentais
  • Array (Vetor): Coleção de elementos contíguos na memória. Acesso por índice é $O(1)$. Inserção/Remoção no meio é $O(n)$ (lento).
  • Lista Ligada (Linked List): Coleção de elementos (nós) onde cada nó aponta para o próximo. Inserção/Remoção é $O(1)$ (rápido). Acesso/Busca é $O(n)$ (lento).
  • Pilha (Stack): Estrutura LIFO (Last-In, First-Out). Operações: push (adicionar ao topo) e pop (remover do topo). Usos: Histórico "Desfazer", chamadas de função do sistema.
  • Fila (Queue): Estrutura FIFO (First-In, First-Out). Operações: enqueue (adicionar ao fim) e dequeue (remover do início). Usos: Fila de impressão, gerenciamento de tarefas.
  • Hash Table (Mapa): Usa uma Função de Hash para mapear uma chave (ex: "nome") a um índice de array. Permite buscas quase instantâneas ($O(1)$ na média). O principal desafio é o tratamento de Colisões (quando duas chaves geram o mesmo índice).
  • Árvore Binária de Busca (BST): Estrutura de nós onde cada nó tem no máximo dois filhos. O filho da esquerda é menor que o pai, e o da direita é maior. Permite buscas, inserções e remoções em $O(\log n)$ se a árvore estiver balanceada.

III. Sistemas Operacionais (SO) e Redes

1. Conceitos de SO
  • Kernel: O núcleo do SO. Gerencia a CPU, memória e dispositivos.
    • Monolítico: Todos os serviços do SO rodam no mesmo espaço de memória (rápido, mas instável, ex: Linux).
    • Microkernel: Apenas o básico roda no kernel; drivers e serviços rodam como processos separados (estável, mas lento).
  • Processo vs. Thread: Um Processo é um programa em execução com seu próprio espaço de memória isolado. Uma Thread é uma unidade de execução dentro de um processo; threads do mesmo processo compartilham memória (mais leve, mas mais complexo de gerenciar).
  • Escalonador (Scheduler): O componente do Kernel que decide qual processo/thread deve usar a CPU em cada momento.
  • Memória Virtual e Paginação: O SO usa o disco (Swap) para simular mais RAM do que existe fisicamente. A memória é dividida em blocos (Páginas) que são movidos entre a RAM e o disco conforme necessário.
  • Deadlock: Uma situação em que dois ou mais processos estão travados, cada um esperando por um recurso que o outro está segurando.
2. Redes de Computadores (Modelo TCP/IP)
  • Camada de Aplicação: Onde os programas do usuário interagem. Protocolos:
    • HTTP/HTTPS: Protocolo da Web.
    • DNS (Domain Name System): Traduz nomes de domínio (ex: google.com) para endereços IP.
    • SMTP: Envio de e-mail. IMAP/POP3: Recebimento de e-mail.
  • Camada de Transporte: Gerencia a comunicação ponta-a-ponta.
    • TCP (Transmission Control Protocol): Confiável, orientado à conexão. Usa o "Aperto de Mão Triplo" (SYN/SYN-ACK/ACK) para estabelecer conexão. Garante a ordem e a entrega dos pacotes. Usado para web, e-mail, arquivos.
    • UDP (User Datagram Protocol): Não confiável, sem conexão. Apenas envia os pacotes (datagramas). Rápido, mas pode perder dados. Usado para streaming, jogos online, DNS.
  • Camada de Rede (Internet): Responsável pelo endereçamento lógico e roteamento. Protocolo: IP (Internet Protocol).

IV. Tópicos Avançados (Bancos de Dados, Segurança, IA)

1. Bancos de Dados
  • Relacional (SQL): Dados em tabelas estruturadas.
    • Normalização: Processo para reduzir a redundância de dados (1FN, 2FN, 3FN).
    • Chaves: Primária (identificador único da linha) e Estrangeira (liga a uma chave primária de outra tabela).
    • SQL (Structured Query Language): SELECT ... FROM ... WHERE ... (Consultar), JOIN (Combinar tabelas).
    • Propriedades ACID: Garantem a confiabilidade das transações (Atomicidade, Consistência, Isolamento, Durabilidade).
  • Não Relacional (NoSQL): Dados flexíveis. Melhor para escalabilidade horizontal. O Teorema CAP afirma que é impossível ter simultaneamente Consistência, Disponibilidade e Tolerância a Particionamento (tem que escolher dois).
2. Segurança Cibernética
  • Tríade CIA: Confidencialidade, Integridade, Disponibilidade.
  • Criptografia Simétrica: A mesma chave secreta é usada para criptografar e descriptografar (ex: AES). Rápida, mas difícil de compartilhar a chave.
  • Criptografia Assimétrica (Chave Pública/Privada): Um par de chaves. O que a chave pública criptografa, só a chave privada descriptografa. É a base do HTTPS e assinaturas digitais.
  • Hashing: Uma função de mão única (irreversível) que gera uma "impressão digital" (hash) dos dados. Usada para verificar a Integridade de arquivos e para armazenar senhas com segurança.
3. Inteligência Artificial e Machine Learning
  • Aprendizado Supervisionado: O modelo aprende com dados previamente rotulados (ex: e-mails marcados como "spam" ou "não spam").
  • Aprendizado Não Supervisionado: O modelo encontra padrões em dados não rotulados (ex: agrupar clientes com comportamentos de compra semelhantes).
  • Aprendizado por Reforço: O modelo aprende tomando decisões e recebendo recompensas ou punições (ex: IA de jogos).

A complexidade $O(n \log n)$ em algoritmos de ordenação como Quick Sort ou Merge Sort reflete a combinação de dois passos: o fator $\log n$ vem do processo de "dividir" o problema (como na Busca Binária), e o fator $n$ vem do processo de "combinar" ou "comparar" os sub-problemas. É a solução ótima para ordenar grandes coleções de dados!
